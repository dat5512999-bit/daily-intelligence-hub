# FAQ

## 為什麼 Live 報告有來源警示？

公開平台可能限流、改版或暫時不可用。系統故意採容錯設計，讓其他來源仍可完成報告。

若全部來源同時失敗，報告會保留來源警示而不會虛構新聞；請稍後重試或先以 `--demo` 確認程式流程。

## 頁首的「來源正常／部分來源受限」是什麼？

這是資料品質提示，不是漂亮用語。至少 3 個來源成功且沒有錯誤才會顯示「來源正常」；仍有 3 個以上來源但某平台失敗，顯示「部分來源受限」；只有 1 到 2 個來源則是「資訊來源不足」。因此，如果又遇到只剩媒體內容，系統會直接提醒你本次資料不完整，而不是偽裝成即時連線正常。

## 為什麼這次只看到媒體／搜尋，沒有 Dcard 或 Reddit？

你截圖裡的原因是 Dcard 暫時拒絕 GitHub Actions 的共用 IP（403），Reddit 暫時限流（429），而 Google News RSS 則有一筆格式不完整。這不是你手機或操作錯誤。

0.9.12 起，單一 RSS、關鍵字或社群頻道失敗時，都不會讓同來源前面成功抓到的內容消失；Dcard 會再試一個公開端點。若 Dcard 仍拒絕，系統會坦白略過，不會以登入、Cookie 或違反平台規則的方式強抓資料。

## 每小時自動更新是在什麼時候？

GitHub Actions 的排程是固定每小時一次，不是從你手動按下的時間開始每 60 分鐘倒數。0.9.12 起安排在每個整點過後約 17 分鐘執行（例如台灣時間 10:17、11:17）；GitHub 偶爾會因排程繁忙延後幾分鐘。執行成功後，GitHub Pages 通常還需要約 1 至 3 分鐘更新。

若超過 25 分鐘仍沒有新的執行紀錄，到 GitHub 專案的 **Actions → Daily Intelligence Report** 看最新一筆；第一次更新此版本後也可以先手動執行一次確認。

## 為什麼報告有時只剩股市或台中新聞？

Live Mode 會依當天公開來源的資料量與熱度排序。若股市或在地突發新聞很多，舊版可能會讓這些內容佔掉大部分版面。

0.9.11 起每個頻道最多先顯示 3 則，並會合併相近標題，盡量保留 AI／Codex、生活流行、遊戲、動漫、活動與市場等不同興趣，不讓單一類型洗版。若某分類當天完全沒有公開資料，系統仍不會硬編內容。

## 為什麼要加入社群冷門雷達？

你的目標不是只看新聞，而是想知道現在年輕人、遊戲圈、動漫圈或生活圈正在紅什麼。有些話題還沒變成新聞，只會先出現在 Dcard、熱搜、短影音標籤或 Reddit 圈內討論。

0.9.3 起，這類來源會進入「現在紅什麼／冷門苗頭」。它代表值得留意，不代表已經被證實；重大事件仍要點進來源確認。

## Dcard 生活類會抓什麼？

0.9.11 起，Dcard 會抓更多熱門貼文，並把美食、旅遊、穿搭、彩妝、感情、閒聊、有趣、梗圖、心情、寵物、居家、汽機車等分類放進「生活流行」。如果標題或摘要同時提到台中與餐廳、活動、展覽、優惠、景點等字眼，會歸到「台中好康與活動」。

0.9.14 起，Dcard 是候補來源。即使 Dcard 拒絕 GitHub 讀取，生活內容仍會由交通部觀光署公開活動、公開趨勢與興趣搜尋補足。

## 小紅書可以查嗎？

目前不直接登入或硬爬小紅書貼文，因為這種做法不穩、容易失效，也比較難判斷真偽。0.9.11 先採用「小紅書 + 台灣流行／餐廳／穿搭／景點／爆紅」等公開搜尋與新聞 RSS 訊號，當作生活趨勢的間接參考。

如果未來要更準，建議走合規資料來源、人工訂閱清單或可公開檢索的第三方趨勢來源，而不是偷抓登入後內容。

## 持股雷達是投資建議嗎？

不是。持股雷達只會把你關注的股票與概念股相關消息拉到前面，幫你知道今天市場或新聞有沒有提到聯電、臻鼎、0050、精材、群創、星宇航空與機器人概念股。

它不會判斷買進、賣出或停損停利；真正交易前仍要回到券商、公司公告、財報與可信財經來源確認。

## 「多看這類／少看這類」會影響下次自動更新嗎？

0.9.6 的回饋先存在同一台手機或瀏覽器，用來立刻清理版面：按「少看這類」會把該分類在目前裝置上隱藏，按「多看這類」會把偏好標記起來。

它目前不會改 GitHub 上的設定檔，也不會跨裝置同步。若要讓下次自動更新真的改變排序，需要下一階段加入安全的偏好同步流程。

## 為什麼有些熱搜或短影音標籤不會出現？

系統會濾掉低上下文、普通人看不懂的短詞或代碼，例如孤立英文字、無意義短標籤。這是為了避免報告出現看了也不知道要幹嘛的內容。

## 為什麼手機主畫面圖示沒有更新？

手機會快取已加入主畫面的網站圖示。0.9.10 起已加入專用 App 圖示與 manifest；若你之前已經加過主畫面，請先刪掉舊捷徑，再用瀏覽器重新「加入主畫面」。

## 是否需要 API Key？

Sprint 1 不需要。摘要採可重現的規則型摘要；未來 Sprint 才會討論可選 LLM 摘要。

## 為什麼 Facebook、Instagram、Threads 還沒有立即加入？

這三個平台的官方資料取得需要 Meta 應用程式、存取權杖與平台權限；它們不適合以不穩定的方式任意抓取公開貼文。系統已保留來源位置，下一階段會在合規前提下設計官方接入流程。

## 「跨平台熱度」等於是真的嗎？

不等於。它只代表多個來源都出現同一話題。系統會將單一社群話題標為「尚未核實」，不應當成事實或直接轉傳；重要事件仍應開啟連結，查看官方公告或可信媒體的原始報導。

## 報告會不會收集我的資料？

不會。Sprint 1 無登入、資料庫、Cookie 或個人偏好記憶。
