# 測試清單

- [ ] 首頁 Hero、Alert、Ranking、Compact List、Timeline 有各自用途，並非所有內容都使用同一種卡片。
- [ ] 圖片卡只使用可由來源安全推導且可用的 YouTube 官方縮圖。
- [ ] RSS 內的 HTML 片段、未翻譯的英文 Reddit／Hacker News 貼文不可出現在首頁。
- [ ] AI、遊戲、動漫、持股、台中、生活與 GitHub 頻道都必須有中文快速入口；沒有可靠資料時須顯示空狀態。
- [ ] 首頁僅先顯示今日必看三件事；分類內容預設收合。
- [ ] 圖片載入失敗後，若沒有其餘圖片卡，整個圖片區必須消失，不可留下破圖或空白大框。

- [ ] `python -m unittest discover -s tests -v` 全數通過。
- [ ] `--demo` 在離線狀態建立 Markdown 與 HTML。
- [ ] `--live` 在一個來源失敗時仍建立報告。
- [ ] preview.html 可直接以瀏覽器開啟。
- [ ] Markdown 連結可點擊且 TOP10 最多十項。
- [ ] 開啟報告後，使用者可在 60 秒內看懂「今天紅什麼」與各分類入口。
- [ ] 生活、遊戲、動漫等頻道若有候選資料，最多顯示 3 則，且相近話題不重複洗版。
- [ ] Dcard 生活類可歸到「生活流行」，台中生活活動可歸到「台中好康與活動」。
- [ ] 單一社群話題不得被標示成已確認事實。
- [ ] 單一 RSS 或單一關鍵字失敗時，其他成功資料仍須保留。
- [ ] 只有 1～2 個來源時，頁首必須顯示「資訊來源不足」，不得顯示「即時連線」。
- [ ] 有來源錯誤但仍有 3 個以上來源時，頁首必須顯示「部分來源受限」。
- [ ] GitHub Actions 工作流程必須保留每小時排程（目前為每小時第 17 分鐘）。
- [ ] 頁首必須顯示「檢查最新頁面」與「手動產生新報告」，後者連到本專案正確的 GitHub Actions 工作流程。
- [ ] 年輕人流行、遊戲、動漫、運動與 GitHub 在候選資料足夠時都必須顯示 3／3 則。
- [ ] 遊戲三則應優先來自不同作品，不得全部被幻獸帕魯或單一作品佔滿。
- [ ] GitHub Trending 的三個不同 repository 不得因共同標題前綴而合併成一則。
