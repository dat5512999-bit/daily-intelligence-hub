# Changelog

## 0.9.17 - 2026-07-11

- 卡片新增「現在看／值得看／有空看／路過知道即可」判斷，協助忙碌使用者決定是否投入時間。
- 卡片與 Markdown 新增資料性質：官方公開資訊、媒體報導、搜尋熱度、社群討論、開發者社群或跨來源整理。
- 新增測試，確認突發事件、官方活動與搜尋趨勢都有正確的判斷與來源標示。

## 0.9.16 - 2026-07-11

- 首頁「今天先看這三件事」與下方分類頻道不再重複顯示同一則情報。
- 分類頻道現在只列出首頁精選以外的新內容，降低忙碌使用者的重複閱讀成本。
- 新增測試，確認前三則精選不會再次出現在分類卡片中。

## 0.9.15 - 2026-07-11

- 修正「多看這類／少看這類」會連動整個分類的設計錯誤。
- 回饋改為「多看這則／少看這則」，只標示或隱藏你實際按下的單張卡片。
- 舊的分類型手機回饋資料會自動清除，避免舊偏好繼續影響新版畫面。

## 0.9.14 - 2026-07-11

- 新增「觀光署活動」官方生活來源：不需 Dcard、帳號或 API Key，提供展覽、旅遊、美食、市集、文化與限時活動公開資訊。
- Dcard 改為候補訊號，不再是手機版生活內容的必要條件。
- 新增來源解析測試，確認會抓活動而非網站導覽連結。

## 0.9.13 - 2026-07-11

- 加入正式驗收資料狀態：`來源正常`、`部分來源受限`、`資訊來源不足`、`資料不足`。
- 修正頁首一律顯示「即時連線」的誤導：當只剩 1～2 個來源或來源有錯誤時，會明確標示目前資料品質。
- Markdown 同步顯示資料狀態與白話說明。
- 新增自動測試，驗證單一來源不會被誤標正常、來源失敗會顯示受限狀態，以及 GitHub 每小時排程仍存在。

## 0.9.12 - 2026-07-11

- 修正 Google News 某一筆 RSS 格式異常時，整個新聞來源一起被丟掉的問題；現在會保留其餘成功資料。
- 個人興趣新聞改為每小時輪替少量查詢，降低免費公開 RSS 的壓力，同時讓遊戲、動漫、生活與持股關鍵字輪流被掃描。
- Reddit 改為少量核心頻道、逐一容錯；被限流時不會讓已抓到的其他 Reddit 內容消失。
- Dcard 新增合法公開端點備援；若 GitHub 共用 IP 被拒絕，會清楚說明並略過，不使用登入、Cookie 或繞過手段。
- GitHub Actions 排程從整點移到每小時第 17 分鐘，降低排程尖峰延遲機率。
- 來源警示改為中文可理解的說明，不再把 HTTP 403／429 或 XML 技術錯誤直接丟給使用者。

## 0.9.11 - 2026-07-11

- 新增「生活流行」頻道，強化 Dcard 美食、旅遊、穿搭、感情、閒聊、有趣等公開熱門討論。
- 新增小紅書間接訊號：透過公開搜尋/新聞關鍵字觀察台灣生活趨勢，不做登入式抓取。
- 頁面新增分類快速入口，手機上可直接跳到生活、遊戲、動漫、股票等頻道。
- 排名改為每個分類最多 3 則，並保留相近內容合併，降低同質新聞洗版。
- 遊戲興趣補強 SF Online、Special Force、地平線/Forza、GTA、魔獸爭霸/Warcraft、寒冰霸權等關鍵字。
- 卡片關聯標籤新增生活苗頭與遊戲雷達，並修正台中突發事件被「周邊」誤判成動漫的問題。

## 0.9.10 - 2026-07-11

- 新增 PWA 手機主畫面設定：`manifest.webmanifest`、`app-icon.svg`、theme color 與 apple touch icon。
- 手機主畫面圖示改為深色霓虹雷達風格，避免只顯示黑底單字。
- GitHub 更新包整合 0.9.7 至 0.9.10 的改動，可一次上傳更新。

## 0.9.9 - 2026-07-11

- 遊戲查詢加強幻獸帕魯/Palworld 1.0、Pocketpair、更新、Game Pass 與卡牌相關訊號。
- Google News 個人興趣查詢每組最多取 8 則，增加遊戲與動漫候選內容。
- 頻道區改為每個分類最多顯示 5 則完整內容，不再只顯示扣除前三名後的剩餘項目。
- Demo Mode 新增多則幻獸帕魯與動漫 IP 範例，方便檢查每個分類不只一張卡片。

## 0.9.8 - 2026-07-11

- 強化短影音趨勢品質過濾，移除像 `#184d0a__a` 這類機器碼、頁面 id 或無上下文標籤。
- TikTok 來源改為只保留人類可讀的中文/日文/韓文標籤或較完整英文標籤。
- 新增測試，確認低意義 TikTok 標籤不會再進入報告。

## 0.9.7 - 2026-07-11

- 報告更新時間改為台灣時間顯示，不再顯示 UTC。
- Markdown 與 HTML 共用台灣時間格式，避免兩種輸出不一致。
- 新增測試，確認報告會顯示「台灣時間」且不再出現 UTC。

## 0.9.6 - 2026-07-11

- 濾除低上下文趨勢字，例如孤立的 `meet` 與無意義短標籤，避免普通使用者看不懂也不想點。
- 報告上限提高到 18 則，每個分類最多 5 則，改善每類只有一張卡片的問題。
- 動漫興趣擴充為 Re:0、無職轉生、轉生史萊姆、影之強者、海賊王、犬夜叉、火影忍者、我獨自升級、實力至上主義、盾之勇者、進擊巨人、鬼滅、宮崎駿與周邊/聖地巡禮/遊戲/電影。
- 手機頁新增「多看這類／少看這類」本機回饋按鈕，可先在同一台裝置清理不感興趣的分類。

## 0.9.5 - 2026-07-11

- GitHub Actions 自動更新頻率從每天一次改為每小時一次。
- 保留手動 `Run workflow`，需要立即更新時仍可在 GitHub Actions 觸發。
- 更新 README 與手機使用說明，標示免費排程可能延遲、不保證準點。

## 0.9.4 - 2026-07-11

- 介面改為更偏效率情報雜誌：新增情報數、苗頭數、持股數與來源數，方便快速判斷今天是否值得深入。
- 視覺從工程控制台微調為更乾淨的質感卡片版型，兼顧工業工程的掃描效率與生活風格美感。
- 股票關注加入聯電、臻鼎、0050、精材、群創、星宇航空與機器人概念股。
- 新增持股雷達提示，股票相關內容會以「和你目前關注的股票或概念股有關」呈現。

## 0.9.3 - 2026-07-11

- 將產品定位從新聞摘要調整為「今天紅什麼情報雷達」：新聞保底，社群與搜尋苗頭優先。
- Dcard、Google 熱門搜尋、TikTok 與 Reddit 改為探索來源，可收進不完全命中既有興趣詞的冷門話題。
- 新增社群冷門雷達、搜尋趨勢、短影音趨勢頻道，並在報告中標示這類內容需要查證。
- Demo Mode 新增生活小情報與搜尋苗頭範例，讓離線預覽更接近真實使用情境。

## 0.9.2 - 2026-07-11
